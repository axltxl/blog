As part of the big changes that are coming for the 0.2 release, I needed in special to implement this one. In order to bring assurance to the code to make it even more portable, I considered necessary to put all integers on a leash because of the "liberties" they have across compilers and operating systems.
/* Integer types */
typedef int64_t q_int64_t;
typedef int32_t q_int32_t;
typedef int16_t q_int16_t;
typedef int8_t q_int8_t;

typedef uint64_t q_uint64_t;
typedef uint32_t q_uint32_t;
typedef uint16_t q_uint16_t;
typedef uint8_t q_uint8_t;

It switches between various implementations of malloc/realloc/calloc/free Use it at your own risk as it is highly experimental, current optional memory allocators are the following:

stdc - Use the Standard C Library implementation
dlmalloc - Doug Lea's Malloc
jemalloc - Jason Evans' Malloc
